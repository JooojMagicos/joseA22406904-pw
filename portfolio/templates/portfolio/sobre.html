Sobre esta Aplicação

1. Arquitetura MVT do Django

O Django segue o padrão MVT — Model, View, Template, uma variante do clássico MVC adaptada ao contexto web. Os três componentes têm responsabilidades bem distintas:

  • Model — Define a estrutura dos dados e comunica com a base de dados via ORM. Cada classe em models.py corresponde a uma tabela na BD.
  • View — Contém a lógica de negócio. Recebe o pedido HTTP, interage com os Models e retorna uma resposta (normalmente renderizando um Template).
  • Template — Ficheiro HTML com a linguagem de template do Django ({% verbatim %}{{ variavel }}{% endverbatim %}, {% verbatim %}{% for %}{% endverbatim %}, etc.) que gera o HTML final enviado ao browser.

Fluxo de informação

  1. O utilizador clica num link ou submete um formulário no browser.
  2. O Django analisa o URL e, através do urls.py, mapeia-o para uma função View.
  3. A View consulta ou altera os dados através dos Models (ORM → SQLite).
  4. A View passa os dados ao Template, que os renderiza em HTML.
  5. O HTML é devolvido ao browser como resposta HTTP.

Modelação em models.py

class TipoTecnologia(models.Model):
    nome = models.CharField(max_length=50, choices=OPCOES, unique=True)
    descricao = models.TextField(blank=True)

class Tecnologia(models.Model):
    tipo = models.ForeignKey(TipoTecnologia, null=True, blank=True, on_delete=models.SET_NULL)
    nome = models.CharField(max_length=100)
    descricao = models.TextField()
    versao = models.CharField(max_length=20, blank=True)

class Projeto(models.Model):
    uc = models.ForeignKey(UnidadeCurricular, on_delete=models.CASCADE)
    tecnologias = models.ManyToManyField(Tecnologia, related_name='projetos')
    titulo = models.CharField(max_length=200)
    descricao = models.TextField()
    ano = models.IntegerField()

class Competencia(models.Model):
    nome = models.CharField(max_length=100)
    tipo = models.CharField(max_length=50)
    descricao = models.TextField()
    tecnologias = models.ManyToManyField(Tecnologia, blank=True)
    projetos = models.ManyToManyField(Projeto, blank=True)
    formacoes = models.ManyToManyField('Formacao', blank=True)

class Formacao(models.Model):
    nome = models.CharField(max_length=200)
    instituicao = models.CharField(max_length=200)
    tipo = models.CharField(max_length=50)
    data_inicio = models.DateField()
    data_fim = models.DateField(null=True, blank=True)

Como usei o Git e o GitHub

Ao longo do desenvolvimento usei o Git para controlo de versões, fazendo commits regulares a cada funcionalidade concluída. O GitHub serviu como repositório remoto e backup do projeto. As principais vantagens foram:

  • Histórico completo de todas as alterações feitas ao código
  • Possibilidade de reverter para versões anteriores em caso de erro
  • Trabalho organizado com commits descritivos por laboratório
  • Facilidade de aceder ao código a partir de qualquer máquina
